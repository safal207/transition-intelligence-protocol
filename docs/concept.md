# Concept

Transition Intelligence Protocol is a framework for understanding decisions as transitions between states.

Most systems focus on outputs:

```text
input -> answer -> action
```

This protocol focuses on the deeper structure:

```text
state -> tension -> cause -> transition -> cooperation -> action
```

## Core thesis

A decision is not complete when an action is selected.

A decision becomes meaningful when the system can explain:

- what state it was in;
- what pressure forced change;
- what cause justified the transition;
- what future state the action may create;
- whether the transition remains stable for all important participants.

## Practical purpose

The protocol can be used to analyze:

- startup decisions;
- human-AI cooperation;
- agentic AI actions;
- family or team conflicts;
- product pivots;
- negotiation states;
- safety boundaries.

## Non-goals

This project is not a fortune-telling system.

It does not claim to know the future. It creates a structured way to reason about possible transitions before action becomes commitment.

## Short definition

Transition Intelligence is the capacity to understand not only what is happening now, but what the situation is becoming, why it is changing, who participates in the change, and what action can be justified.
